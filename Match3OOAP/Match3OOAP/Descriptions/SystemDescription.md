### Task 1 - Предварительное описание системы

- Игровой интерфейс - отображает игровое поле и элементы, отображает счёт, отображает информацию о действиях игрока, 
   отображает опции применения бонусов и перезапуска игры, предоставляет поле ввода. 
- Обработчик ввода игрока - обрабатывает введенные через интерфейс команды и передаёт в интерфейс результат выполнения команд. 
- Контроллер игрового процесса - запуск, перезапуск и завершение игры.
- Игровая логика - перестановка элементов, проверка перестановки, генерация и размещение элементов (падение элементов вниз), удаление элементов.
- Подсчёта статистики - реагирует на события игровой логики, ведёт подсчёт количества очков, ведёт журнал ходов.
- Игровой бонус - вычисляет, хранит и предоставляет текущий бонус.


### Task 2 - Границы разрабатываемой системы

#### Входит в систему

- Логика игрового процесса.
- Обработка ввода.
- Обработка консольного вывода.
- Система бонусов.
- Система игровой статистики.

#### Не входит в систему

- Полноценный графический интерфейс.
- Сохранение состояния матча.
- Сохранение итогов матча.

#### Главные подсистемы

- Подсистема игрового состояния — хранение и изменение состояния поля и очков.
- Подсистема игровой логики — проверка ходов, поиск комбинаций, применение правил.
- Подсистема обработки ввода - правила преобразования пользовательского ввода в игровые команды.
- Подсистема отображения - показывает текущее состояние игрового процесса.

#### Пользовательские метафоры
_(что именно пользователь/заказчик понимает под тем, что в рамках проекта он называет, например, "Автомобиль" или "Товар" или "Клиент")_

- Элемент - игровой предмет, который располагается на игровом поле и перемещается игроком для составления комбинаций.
- Сетка - игровое поле фиксированного размера на котором размещаются элементы.
- Комбинация - сочетание трёх и более элементов одного типа расположенных непрерывной последовательностью по горизонтали или вертикали.
- Инициализация - создание игровой сетки фиксированного размера и заполнение её случайными элементами.
- Ход - действия игрока связанные с перемещением элементов или применением бонуса и дальнейшей обработкой игровой логикой этих действий.
- Генерация элементов - создание новых элементов и заполнение ими игрового поля сверху вниз.
- Ввод координат - выбор клеток поля для перестановки.
- Бонус - награда за сложные комбинации в игре.
- Применение бонуса - модификация игрового процесса позволяющая заработать больше игровых очков.
- Игровые очки - награда за создание комбинаций. Показатель количества и качества созданных комбинаций.
- История ходов - упорядоченный по очерёдности событий список, содержащий информацию о каждом ходе (клетки с которыми взаимодействовали
  и результат хода).
- Общая статистика - общее количество очков и количество сделанных ходов за один матч.

#### Функциональность

- Запуск матча.
- Отображение игрового процесса.
- Взаимодействие с игровым процессом.
- Начисление бонусов.
- Применение бонусов.
- Завершение матча.
- Перезапуск матча.
- Ведение истории матча.
- Подсчёт статистики матча

#### Библиотеки повторного использования

- Библиотека работы с консолью.
- Библиотека работы с коллекциями.
- Библиотека работы с текстом.
- Библиотека генерации случайных чисел.

### Task 3 - Классы анализа

- Grid - сетка для размещения элементов.
- Element - игровой элемент определённого типа.
- Combination - комбинация упорядоченных элементов.
- Bonus - награда за комбинацию.
- GameplayModification - модификация игрового процесса, дополняет игровую логику новыми действиями или модифицирует начисление очков.
- Move - ход игрока, перестановка элементов на поле.
- Score - счёт игрока за игровой матч.
- MoveHistory - история ходов за игровой матч.
- GameUI - отображение состояния игры.
- InputHandler - обработчик ввода.


### Task 4 - Классы реализации

Остальное реализуется стандартными библиотеками.


### Task 5 - Классы проектирования

- GameLogicFacade - фасад игровой логики, единая точка входа для работы с игровой логикой (сделать ход, получить результат хода,
  узнать счёт, узнать доступный бонус, узнать историю ходов).
- RandomElementsGenerator - генератор случайных элементов.
- GameRoot - корневой класс запуска и поддержания обновлений игрового процесса (Composition root).
- GameStrategy - описывает порядок действий для приведения игры в нужно состояние (матч готов к запуску, матч запущен, матч завершён).
- GameStateController - управляет состоянием игры (запускает, перезапускает, завершает).
- GuiView - gui интерфейс для отображения состояния игры.
- GuiPresenter - преобразовывает команды из gui в команды управления игрой и выводит в gui результат команд.
- ElementGravity - контроллер падения элементов, управляет тем, как падают на поле элементы после удаления и генерации (похож на класс реализации т.к. содержит логику падения элементов, а не описывает конкретный тип данных).
- MoveStrategy - стратегия хода (по сути контроллер), определяет порядок действий для обработки хода и выдачи результата.
- BonusCalculator - рассчитывает бонус исходя из комбинации (похож на класс реализации т.к. содержит логику расчёта бонуса, а не описывает конкретный тип данных).


### Task 6 - Отбракованные классы

- Element - используется как значение, не имеет собственных операций, не является АТД. Операции с элементами являются частью Grid и Combination.
- Bonus - также, как и элемент не имеет собственных операций, не является АТД. Обязанности распределены между АТД игровой логики.
- GameplayModification - действие, описывает процесс модификации игры. 
- InputHandler - обозначает действие, обработку ввода, не является АТД. Его обязанности принадлежат GuiPresenter.
